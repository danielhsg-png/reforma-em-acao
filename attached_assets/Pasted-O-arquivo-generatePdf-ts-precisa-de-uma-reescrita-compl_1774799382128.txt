### PROBLEMAS CRÍTICOS A CORRIGIR (bugs confirmados no PDF atual)

BUG 1 — Caracteres inválidos:
O jsPDF com fontes padrão (Helvetica) NÃO suporta Unicode, emojis ou caracteres
especiais como ✅ !' '& þ → ÷ etc.
Crie uma função auxiliar sanitizeText(str: string): string que percorre a string
e substitui/remove qualquer caractere com charCode > 255.
Substituições obrigatórias em TODO o código:
- ✅ / ☑ / þ  →  remover
- 🔔 / !' / &  →  "->"
- → (seta Unicode)  →  "->"
- ÷  →  "/"
- aspas curvas  →  aspas retas
- qualquer emoji  →  remover

BUG 2 — Texto cortado nas margens:
Todo texto longo DEVE passar por splitTextToSize() antes de renderizar.
Largura máxima: pageWidth - marginLeft - marginRight = 170mm.
Após splitTextToSize(), avançar y: yPos += linhas.length * lineHeight

BUG 3 — Overflow de página:
Antes de cada bloco, verificar: if (y + alturaEstimada > pageHeight - margin) addPage()
Criar função auxiliar: checkPageBreak(doc, y, estimatedHeight, margin): number

BUG 4 — Logo sobreposto na capa:
Usar apenas texto com fontes jsPDF na capa, sem imagem/SVG posicionada por cima.
"REFORMA" em Helvetica bold 32pt + "EM ACAO" em Helvetica bold 32pt, laranja,
calculando larguras para centralizar os dois juntos na mesma linha.

BUG 5 — Fonte itálica nas ações:
Remover uso de Courier ou italic em todo o documento.
Usar APENAS Helvetica normal e Helvetica bold.
